# 📈 quantsim

**A full quant trading stack in one package.** Research a strategy with Monte Carlo simulation, backtest it on real market data with order-book-based execution costs, then deploy the *identical strategy object* to a live paper-trading bot that commits its P&L into git. One `Strategy` class flows through the entire lifecycle — zero reimplementation drift between what you tested and what trades.

```
        research                 backtest                    deploy
  ┌────────────────┐      ┌──────────────────┐      ┌────────────────────┐
  │  quantsim mc   │      │ quantsim backtest │      │  quantsim live run │
  │  GBM · VaR     │ ───▶ │ real data · costs │ ───▶ │  same Strategy,    │
  │  CVaR · paths  │      │ via order book    │      │  Alpaca paper API  │
  └────────────────┘      └──────────────────┘      └────────────────────┘
                                   ▲
                     ┌─────────────┴──────────────┐
                     │     quantsim.orderbook     │
                     │ price-time priority engine │
                     └────────────────────────────┘
```

Built on NumPy (the only dependency), 36 tests, MIT licensed.

## 🚀 Quickstart

```bash
pip install "quantsim[all] @ git+https://github.com/hilothefunnydog123-coder/quantsim.git"

# 1) backtest a 20/100 SMA crossover on 10 years of real SPY data
quantsim backtest --symbol SPY --strategy sma --plot spy.png

# 2) same backtest, but every trade pays realistic market impact by
#    walking a synthetic limit order book instead of a flat fee
quantsim backtest --symbol SPY --strategy sma --execution book

# 3) paper-trade the same strategy (local simulator — zero setup)
quantsim live run --symbol SPY --strategy sma
quantsim live status
quantsim live report --out docs/report.html

# 4) Monte Carlo: 10,000 possible futures for a $10k portfolio
quantsim mc --initial 10000 --mu 0.07 --sigma 0.18 --years 10
```

Data comes from **yfinance** if installed, else **Stooq's free CSV endpoint** (stdlib only, no API key). Any local `Date,Close` CSV works via `--csv`.

## 🧩 One Strategy class, entire lifecycle

```python
import numpy as np
from quantsim import Strategy, fetch, run_backtest, run_cycle, PaperBroker, BookExecution

class Breakout(Strategy):
    name = "breakout"
    warmup = 55

    def target_weight(self, closes: np.ndarray) -> float:
        return 1.0 if closes[-1] >= closes[-55:].max() else 0.0

series = fetch("QQQ", start="2015-01-01")

# backtest it — with market impact from the order book
result = run_backtest(series.closes, Breakout(), dates=series.dates,
                      execution=BookExecution(level_shares=10_000))
print(result.metrics["sharpe"], result.metrics["max_drawdown"])

# deploy it — the same object, no rewrite
broker = PaperBroker("state/portfolio.json")
run_cycle(broker, series.closes, "QQQ", Breakout())
```

For stateful strategies, `run_cycle` replays history exactly like the backtester, so the live weight is **bit-for-bit the backtest's final weight** — there's a test asserting it.

Built-ins: `buyhold`, `sma`, `momentum`, `meanrev` — all deliberately readable; fork them.

## 📊 The matching engine

`quantsim.orderbook.OrderBook` is a real price-time priority limit order book — the data structure inside every exchange. Limit and market orders, FIFO queues per price level, partial fills at maker price, cancels, depth snapshots, deterministic sequence numbers. **~165k orders/sec** in pure Python (`python scripts/bench_orderbook.py`), with randomized invariant tests proving the book never crosses itself and every share is filled, resting, or explicitly discarded.

It powers `BookExecution`: instead of guessing "1 bp per trade", backtests can *execute* each rebalance as a market order against synthetic book liquidity. Small orders pay ~half the spread; big orders eat through levels and pay real impact. Thin out `--level-shares` and watch a churny strategy's edge disappear — that's the lesson most backtests hide.

## 🤖 The live bot (free, scheduled, auditable)

`quantsim live run` does one idempotent cycle: fetch closes → compute weight → rebalance to target. Brokers: a local simulator (default) and **Alpaca's paper API** via plain `urllib` (`--live`, keys in `APCA_API_KEY_ID`/`APCA_API_SECRET_KEY`). The included [GitHub Actions workflow](.github/workflows/trade.yml) runs it every US market morning and **commits the state + HTML P&L report into the repo** — the track record is the git history, not a screenshot. Enable GitHub Pages on `/docs` and your equity curve is a self-updating public URL.

> Paper money only. The Alpaca integration points at `paper-api.alpaca.markets`; this is a research and learning tool, not investment advice.

## 🛡️ The engine is honest

- **No lookahead — enforced by test.** The weight chosen with data through day *t* earns the return from *t* to *t+1*. A test records exactly what each strategy call was allowed to see.
- **Costs on by default** — flat bps or full order-book impact; churn pays either way.
- **Buy & hold always printed side-by-side.** If the strategy doesn't beat it after costs, quantsim tells you first.
- **Statistical tests** — simulated GBM means must converge to the analytical `E[S_T] = S₀·e^(μT)`.
